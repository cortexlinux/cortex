# System Role Management

Manage system personalities and receive tailored package recommendations based on your workstation's specific purpose, powered by autonomous AI detection.

## Overview

Cortex utilizes an AI-first approach to understand the context of your Linux environment. Instead of relying on static rules, Cortex acts as a sensing layer—identifying existing software signals, hardware capabilities, and shell activity patterns—and leverages an LLM to infer the most appropriate system role contextually.

The `cortex role` command group handles the factual gathering of system context and secure persistence of AI-driven classifications.

## Usage

### Basic Commands

```bash
# Auto-detect your system role using AI analysis of local context and patterns
cortex role detect

# Manually set your system role to receive specific AI recommendations
cortex role set ml-engineer

# View help for the role command group
cortex role --help
```

## Features

### 1. Intelligent Context Gathering (Sensing Layer)

Cortex scans your system `PATH` for signature binaries (like `docker`, `gcc`, or `terraform`) and checks for NVIDIA hardware acceleration. It also analyzes your recent shell history to understand your active workflow.

### 2. Pattern Learning

By analyzing recent command patterns (e.g., `pip install`, `pytest`, `docker-compose`), Cortex allows the LLM to "learn" your repetitive behaviors and recommend specific optimizations or aliases to improve your productivity.

### 3. Dynamic Custom Roles

Since recommendations are generated by an AI Architect rather than a hardcoded JSON file, Cortex supports any role you declare (e.g., `minecraft-admin`, `security-researcher`, `data-scientist`) with zero manual configuration.
